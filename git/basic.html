To push your code into the master branch of the git@example.com:example/test.git, you will need to follow several steps. Here are the detailed instructions:

  1. Initializing a local git repository: If you don't already have a local git repository for your project, you must create one. To do this, go to the folder with your code via the command line and run the following command: git init
  2. Adding and committing your code: Add all files and changes in your project to the git index and commit. Use the following commands: git add .
    git commit -m "Your commit comment"
  3. Adding a Remote Repository: Now you need to add the remote repository git@example.com:example/test.git to your local git repository. Run the command: git remote add origin git@example.com:example/test.git
  4. Pushing code to a remote repository: Upload your code to the remote repository master branch: git push -u origin master This command will push your code to the master branch of the remote repository. After this, you will be asked to enter credentials (username and password) to access the remote repository.
  5. Completion: Your code should now be uploaded to the master branch of the git@example.com:example/test.git. You can check this by going to the repository page on GitLab or using the  git log  command to view the commit history.

That's it, your code is now available in the repository on the master branch.
